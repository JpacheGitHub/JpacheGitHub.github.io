使用Hexo博客时一些错误整合

2016-05-08 • 标签: hexo

这里会整合一些在使用 Hexo 博客是的一些错误, 尽可能详细的提供解决方法.

1. { [Error: Cannot find module ‘./build/Release/DTraceProviderBindings’] code: ‘MODULE_NOT_FOUND’ }

 
 
      之前上传新文章的时候粘错了命令, 重新更新了 Hexo, 不愿意等, 所以到一半就 ctrl + c 了, 结果发现再使用的时候出现了个令人发指的错误, 倒是没有耽误使用, 不过每次不管用什么命令, 这个错误都会先出来溜一圈, 很不爽, 大概的错误如下

      具体什么原因现在还想的不太明白, 大概意思就是少了个依赖库, Google 一下发现很多人都犯这个毛病, 大概的解决方法也就那么几种, 几种方法都有的人回帖说好用了,也有一些人回帖说还是不好用, 发这篇博客之前很不幸, 我属于后者. 废话不多说, 上解决方案. 每种方法都有可能不起作用, 一个一个尝试下, 总有一款适合你, 选择了 Hexo, 就要敢于折腾.

1.有人说这是因为天朝的网络原因, 有几个依赖库在安装的时候没有安装成功, 或者中途被取消安装了. 和我的情况吻合, 给了解决方案.

            2.有人说一句不行, 下面这几句都执行了就好了

2.有人这么解释这个错误, 这不就是少了个库么. 然后搜了一下打印出的错误中的 ‘DTraceProviderBindings’ 这个, 发现在博客根目录/node_modules/dtrace-provider 这个目录中出现. 那就重新安装一下. 或者先删掉, 再重新装.

      然后还是不好用…然后我以为是不是原来的那个根目录里面有一些文件已经损坏了, 是不是需要重新建一个目录, 一顿折腾, hexo v 之后我又看见了那可恶的三行错误信息…呵呵嗒…

3.后来有人告诉我说前两个原因都有, 但是因为现在 hexo 更新之后还有不一样的地方和原来, 所以重新安一遍, 弄个新目录就好了. 我就索性把 node.js 和 hexo 都重新安了一遍.
第一次又是以失败告终. 然后我再一个隐藏文件夹里发现是我没有删干净. 重新删, 重新安装, 新建目录, OK, 不用再重复前两个解决方案, 三行错误没有了.
隐藏文件夹是: /usr/local/lib/node_modules/ 目录下的 hexo 文件夹
还有: /usr/local/bin/ 下的 hexo 文件. 直接删掉就好了.
